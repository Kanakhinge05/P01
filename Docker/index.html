Git Version Control

Git is a distributed version control system that tracks changes in source code during software development. It enables collaboration, maintains history, and allows you to work on multiple features simultaneously.

🔀

Core Concepts

  • Repository - Storage for your project and its history
  • Commit - Snapshot of your changes
  • Branch - Independent line of development
  • Merge - Combine changes from different branches
  • Remote - Version of your repo hosted elsewhere
⚡

Essential Commands

git init
Initialize a new Git repository
git clone <url>
Clone a remote repository
git add .
Stage all changes for commit
git commit -m "message"
Commit staged changes with message
🌳

Branching & Merging

git branch <name>
Create a new branch
git checkout <branch>
Switch to a different branch
git merge <branch>
Merge branch into current branch
git pull
Fetch and merge remote changes
🚀

Remote Operations

git remote add origin <url>
Connect to a remote repository
git push origin <branch>
Push commits to remote
git fetch
Download remote changes
git status
Check working tree status

Git Workflow

1. Clone or Initialize

Start by cloning an existing repository or initializing a new one in your project directory.

2. Create a Branch

Create a new branch for your feature or bug fix to keep your work isolated from the main codebase.

3. Make Changes

Write code, edit files, and make your changes. Use git status to track what's been modified.

4. Stage & Commit

Stage your changes with git add, then commit them with a descriptive message about what you changed.

5. Push to Remote

Push your branch to the remote repository so others can see your work and collaborate.

6. Create Pull Request

Open a pull request to merge your changes into the main branch after review and approval.

GitHub Platform

GitHub is the world's leading platform for version control and collaboration. It provides a web-based interface for Git repositories along with powerful features for team collaboration, code review, and project management.

🔗

Collaboration Features

  • Pull Requests - Propose and review code changes
  • Issues - Track bugs and feature requests
  • Projects - Organize and prioritize work
  • Discussions - Community conversations
  • Code Review - Inline comments and suggestions
⚙️

GitHub Actions

Automate your workflow with CI/CD pipelines directly in GitHub. Build, test, and deploy your code automatically on every push or pull request.

  • Automated testing on every commit
  • Deploy to production on merge
  • Schedule periodic tasks
  • Custom workflows with YAML
🛡️

Security & Access

  • Branch Protection - Enforce review rules
  • Security Alerts - Vulnerability detection
  • Dependabot - Automated dependency updates
  • Code Scanning - Find security issues
  • Secret Scanning - Detect exposed credentials
📦

Advanced Features

  • GitHub Pages - Host static websites
  • GitHub Packages - Package hosting
  • Codespaces - Cloud development environment
  • Copilot - AI-powered code suggestions
  • Wiki - Project documentation

Git vs GitHub

Feature Git GitHub
Type Version control system (software) Web-based hosting platform
Purpose Track and manage code changes locally Host repositories and enable collaboration
Installation Installed on your computer Cloud-based service, accessed via browser
Key Features Commits, branches, merges, version history Pull requests, issues, actions, code review
Usage Command-line or GUI tools Web interface + Git commands
Collaboration Local version control Team collaboration and social coding

Best Practices

✅

Commit Best Practices

  • Write clear, descriptive commit messages
  • Make small, focused commits
  • Commit early and often
  • Use present tense in messages
  • Review changes before committing
🌿

Branch Strategy

  • Use descriptive branch names
  • Keep main/master branch stable
  • Create feature branches for new work
  • Delete merged branches
  • Rebase for clean history
🔒

Security Practices

  • Never commit sensitive data
  • Use .gitignore effectively
  • Enable two-factor authentication
  • Review code before merging
  • Keep dependencies updated
🤝

Team Collaboration

  • Use pull requests for code review
  • Document your code and changes
  • Communicate with your team
  • Follow project conventions
  • Test before pushing changes
